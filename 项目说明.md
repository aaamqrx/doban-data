# 豆瓣电影爬虫项目说明

## 一、项目简介

本项目用于批量抓取豆瓣电影数据，并围绕“中国地区电影（大陆 / 香港 / 台湾）”构建一套可重复执行的数据采集流程。当前流程覆盖以下 **6 个阶段**：

1. 按电影类型抓取豆瓣榜单原始数据
2. 从原始数据中筛选中国地区电影
3. 批量下载电影详情页 HTML 到本地缓存
4. 抓取并缓存电影上映前 7 天的百度指数日级结果
5. 解析百度指数缓存并生成中间特征表
6. 解析 HTML 并与基础表合并生成最终总表

最终产出为标准化 Excel 数据文件，可作为后续建模、分析或预测任务的数据基础。

## 三、项目功能说明

### 1. `config.py`

用于统一管理项目配置，主要包含：

- 电影类型与豆瓣 `type_id` 映射
- 中国地区关键词（大陆 / 香港 / 台湾）
- 分页大小
- HTML 下载延时与风控退避参数
- 百度指数 Cookie、地区、重试与预警阈值
- 数据目录与缓存目录
- 豆瓣请求头与 Cookie

说明：

- `TARGET_CHINA_COUNT` 目前作为目标配置保留在 `config.py` 中，但当前代码并没有强制按该值截断或补足数据。
- 百度 Cookie 和豆瓣 Cookie 分开配置，互不影响。

---

### 2. `src/1_get_raw_data.py`

功能：按类型抓取豆瓣榜单接口数据，并保存原始结果。

主要逻辑：

- 遍历 `config.MOVIE_TYPES`
- 对每个类型遍历豆瓣评分区间：
  - `100:90`
  - `90:80`
  - `80:70`
  - `70:60`
  - `60:50`
  - `50:40`
  - `40:30`
  - `30:20`
  - `20:10`
  - `10:0`
- 对每个评分区间继续分页抓取
- 将所有评分区间结果汇总
- 优先按 `subject_id` 去重
- 输出到 `data/raw/<类型>.xlsx`

输出字段包括：

- `subject_id`
- `电影名`
- `评分`
- `评价人数`
- `主演`
- `制片国家/地区`
- `类型`
- `上映时间`
- `详情链接`

---

### 3. `src/2_filter_china.py`

功能：从原始 Excel 中筛选中国地区电影。

主要逻辑：

- 读取 `data/raw/*.xlsx`
- 根据 `制片国家/地区` 判断是否包含中国地区关键词（大陆 / 香港 / 台湾）
- 保留 `subject_id`、`评分`、`评价人数` 等第1步新增字段
- 基于 `上映时间` 生成 `是否春节档`、`是否国庆档`、`是否五一档`、`是否暑期档`、`是否普通周末`
- 节假日档期通过 `chinesecalendar` + 向前侦测 3 天识别节前抢跑影片
- 为结果增加 `影片类型` 列
- 删除原始 `类型` 列，避免与 `影片类型` 重复
- 输出到 `data/filtered/china_<类型>.xlsx`

---

### 4. `src/3_save_html.py`

功能：批量下载中国地区电影详情页 HTML，并缓存到本地。

主要逻辑：

- 读取 `data/filtered/china_*.xlsx`
- 优先使用表内已有的 `subject_id`
- 仅在 `subject_id` 缺失时才从 `详情链接` 中兜底提取
- 请求豆瓣详情页
- 保存到 `html_cache/<类型>/`
- 如果文件已存在则跳过，支持断点续传
- 下载后校验页面是否为有效的豆瓣电影详情页

当前 HTML 文件名规则：

- 优先使用：`subject_id_电影名.html`

当前脚本还包含以下保护：

- 页面标题校验
- 详情结构校验
- 验证码 / 异常请求 / 页面不存在 / 禁止访问 等异常页面拦截
- 请求日志输出（URL、状态码、失败原因）
- 命中风控页时自动退避重试，并在连续拦截时暂停或中止

---

### 5. `src/4_save_baidu_index.py`

功能：抓取并缓存每部电影上映前 7 天的百度指数日级结果。

主要逻辑：

- 读取 `data/filtered/china_*.xlsx`
- 优先使用 `subject_id`，缺失时才从 `详情链接` 兜底提取
- 复用 `utils_dates.py` 中的 `parse_release_date()` 解析上映日期
- 复用 `utils_baidu.py` 中的查询词清洗与时间窗口函数
- 用 `qdata` 请求百度指数数据
- 保存到 `baidu_cache/<类型>/subject_id_电影名.json`

缓存 JSON 中主要保留：

- `subject_id`
- `电影名`
- `百度指数查询词原始`
- `百度指数查询词清洗后`
- `上映日期`
- `百度指数起始日期`
- `百度指数结束日期`
- `百度指数日值列表`
- `百度指数状态`
- `百度指数错误信息`

说明：

- 缓存的不是百度原始加密响应，而是规范化后的日值结构；
- 同时保留原始查询词和清洗后查询词，便于调试；
- 若请求失败，会写入 `error` 状态缓存；
- 若上映日期无法解析，会写入 `missing_release_date` 状态缓存。

---

### 6. `src/5_parse_baidu_index.py`

功能：只读取本地百度缓存，生成真正用于建模的中间表。

主要逻辑：

- 读取 `baidu_cache/<类型>/*.json`
- 对应读取 `data/filtered/china_*.xlsx`
- 优先按 `subject_id` 匹配缓存
- 根据 `百度指数日值列表` 计算 `上映前7天百度搜索指数均值`
- 输出到 `data/baidu/china_*.xlsx`

新增字段包括：

- `上映前7天百度搜索指数均值`
- `百度指数状态`
- `百度指数错误信息`
- `百度指数查询词原始`
- `百度指数查询词清洗后`
- `百度指数样本天数`
- `百度指数非空天数`

解析规则：

- 0、`未收录` 等统一按数值 `0` 参与均值计算；
- 若 7 天都为 0 且请求成功，状态记为 `zero_valid`；
- 若上映日期无法解析，状态为 `missing_release_date`，均值为空；
- 若缓存不存在或请求失败，均值为空，并保留错误状态与错误信息。

---

### 7. `src/6_parse_html.py`

功能：解析本地 HTML 提取补充字段，并与百度增强后的基础表合并。

主要逻辑：

- 遍历 `html_cache/` 下全部类型目录
- 使用 BeautifulSoup 解析 HTML
- 提取以下字段：
  - `subject_id`
  - `电影名`
  - `导演`
  - `片长`
  - `剧情简介`
  - `IMDb编号`
- 读取 `data/baidu/china_*.xlsx`
- 优先直接使用基础表中已有的 `subject_id`
- 只有在 `subject_id` 缺失时，才从 `详情链接` 兜底提取
- 优先按 `subject_id` 左连接合并
- 如果个别记录没有 `subject_id`，再退回按 `电影名` 合并
- 输出最终文件 `data/final/china_all.xlsx`

## 四、完整执行流程

请按如下顺序执行：

```bash
python src/1_get_raw_data.py
python src/2_filter_china.py
python src/3_save_html.py
python src/4_save_baidu_index.py
python src/5_parse_baidu_index.py
python src/6_parse_html.py
```

流程说明：

1. 第1步抓取榜单原始数据
2. 第2步筛选中国地区电影并生成档期特征
3. 第3步下载详情页 HTML 并缓存
4. 第4步抓取百度指数并缓存日值 JSON
5. 第5步从百度缓存生成均值特征中间表
6. 第6步解析 HTML 并生成最终总表

## 八、项目特点

本项目相比旧版脚本，具有以下改进：

1. **结构更清晰**：按配置、脚本、数据、缓存分目录组织
2. **支持多类型抓取**：不再局限于单一类型
3. **评分区间遍历更完整**：第1步自动遍历 `100:90` 到 `10:0`
4. `subject_id`、`评分`、`评价人数`、档期布尔特征都在前两步直接结构化生成
5. **HTML 命名更稳定**：优先使用 `subject_id_电影名.html`
6. **百度指数可复现**：第4步先缓存日值，第5步再解析特征
7. **合并更准确**：最终步骤优先按 `subject_id` 合并
8. **便于维护**：所有核心配置集中在 `config.py`
9. **流程标准化**：六步脚本职责明确，便于逐步执行和排错

## 九、注意事项

1. **Cookie 可能过期**  
   若抓取报错或结果异常，优先检查 `config.py` 中的豆瓣或百度 Cookie 是否失效。

2. **豆瓣榜单接口并非全站全量电影库**  
   第1步基于豆瓣分类榜单接口抓取，得到的数据仍受该接口返回范围限制。

3. **第2步档期布尔特征依赖 `chinesecalendar`**  
   若运行时报模块缺失，请先安装依赖。

4. **第4步依赖 `qdata`**  
   若运行百度指数脚本时报模块缺失或接口异常，请先确认 `qdata` 已安装且当前环境可导入。

5. **0 值不等同于失败**  
   第5步会把成功抓到但 7 天全为 0 的记录标记为 `zero_valid`，这表示低热度，而不是请求失败。

6. **标题清洗会影响查询词**  
   第4步会优先按分隔符截断副标题，缓存中保留原始词和清洗后词，便于排查。

7. **Excel 文件不要在运行时打开**  
   如果 `data/raw/*.xlsx`、`data/filtered/*.xlsx`、`data/baidu/*.xlsx` 或最终输出文件正在被 Excel 占用，脚本可能无法写入。

8. **HTML 与百度步骤耗时较长属于正常现象**  
   因为都包含随机延时、批次暂停和失败重试，用于降低风控风险。

9. **建议抽样检查最终结果**  
   建议至少抽样检查几部电影，确认 `上映前7天百度搜索指数均值`、`导演`、`片长` 等字段正常。

## 十一、总结

本项目已经从原本的若干单用途脚本，升级为一套结构清晰、可重复执行、支持多类型抓取与特征扩展的豆瓣电影采集流程。通过 `config.py + src/六步脚本 + data/html_cache/baidu_cache` 的结构，可以更方便地完成数据抓取、筛选、缓存、解析和汇总，并通过 `subject_id` 优先合并、百度指数先缓存后解析等方式，进一步提升数据完整性、可追溯性与稳定性。
