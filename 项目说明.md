# 豆瓣电影爬虫项目说明

## 一、项目简介

本项目用于批量抓取豆瓣电影数据，并围绕“中国地区电影（大陆 / 香港 / 台湾）”构建一套可重复执行的数据采集流程。当前流程覆盖以下 4 个阶段：

1. 按电影类型抓取豆瓣榜单原始数据
2. 从原始数据中筛选中国地区电影
3. 批量下载电影详情页 HTML 到本地缓存
4. 解析 HTML 提取导演、片长，并与基础数据合并

最终产出为标准化 Excel 数据文件，可作为后续建模、分析或预测任务的数据基础。

## 三、项目功能说明

### 1. `config.py`

用于统一管理项目配置，主要包含：

- 电影类型与豆瓣 `type_id` 映射
- 中国地区关键词（大陆 / 香港 / 台湾）
- 分页大小
- 随机请求延时
- 数据目录与缓存目录
- 请求头与 Cookie

当前支持的电影类型：

- 喜剧：`24`
- 动作：`5`
- 爱情：`13`

这样做的好处是：后续若需要新增类型、调整路径或替换 Cookie，只需要修改这一个文件。

说明：

- `TARGET_CHINA_COUNT` 目前作为目标配置保留在 `config.py` 中，但当前代码并没有强制按该值截断或补足数据。

---

### 2. `src/1_get_raw_data.py`

功能：按类型抓取豆瓣榜单接口数据，并保存原始结果。

主要逻辑：

- 遍历 `config.MOVIE_TYPES`
- 对每个类型遍历豆瓣评分区间：
  - `100:90`
  - `90:80`
  - `80:70`
  - `70:60`
  - `60:50`
  - `50:40`
  - `40:30`
  - `30:20`
  - `20:10`
  - `10:0`
- 对每个评分区间继续分页抓取
- 将所有评分区间结果汇总
- 按 `详情链接` 去重
- 输出到 `data/raw/<类型>.xlsx`

为什么要遍历 `interval_id`：

豆瓣榜单接口一次只能返回某个评分区间的数据。如果只请求单个区间，例如 `20:10`，拿到的只是该评分段内的电影，而不是该类型的完整榜单数据。因此当前代码会把 `100:90` 到 `10:0` 的区间全部遍历一遍，再统一去重。

输出字段包括：

- `subject_id`
- 电影名
- `评分`
- `评价人数`
- 主演
- 制片国家/地区
- 类型
- 上映时间
- 详情链接

额外说明：

- 保存 Excel 前会打印即将写入的路径和记录数。
- 如果 Excel 文件正在被本地打开，保存阶段可能失败，需要先关闭对应文件后再运行。

---

### 3. `src/2_filter_china.py`

功能：从原始 Excel 中筛选中国地区电影。

主要逻辑：

- 读取 `data/raw/*.xlsx`
- 根据 `制片国家/地区` 判断是否包含中国地区关键词（大陆 / 香港 / 台湾）
- 保留 `subject_id`、`评分`、`评价人数` 等第1步新增字段
- 为结果增加 `影片类型` 列
- 删除原始 `类型` 列，避免与 `影片类型` 重复
- 输出到 `data/filtered/china_<类型>.xlsx`

该步骤的作用是把后续要重点处理的中国地区电影数据单独整理出来。

筛选后的文件字段更聚焦，只保留当前流程真正需要的内容。

---

### 4. `src/3_save_html.py`

功能：批量下载中国地区电影详情页 HTML，并缓存到本地。

主要逻辑：

- 读取 `data/filtered/china_*.xlsx`
- 优先使用表内已有的 `subject_id`
- 仅在 `subject_id` 缺失时才从 `详情链接` 中兜底提取
- 请求豆瓣详情页
- 保存到 `html_cache/<类型>/`
- 如果文件已存在则跳过，支持断点续传
- 下载后会校验页面是否为有效的豆瓣电影详情页

当前 HTML 文件名规则：

- 优先使用：`subject_id_电影名.html`
- 例如：`34950184_困兽.html`

这样做的意义是：

- 避免每次解析时重复请求网页
- 降低网络请求次数
- 中途中断后可继续下载
- 通过 `subject_id` 降低重名电影带来的覆盖或误判风险

当前脚本还包含以下保护：

- 页面标题校验
- 详情结构校验
- 验证码 / 异常请求 / 页面不存在 / 禁止访问 等异常页面拦截
- 请求日志输出（URL、状态码、失败原因）
- 命中风控页时自动退避重试，并在连续拦截时暂停或中止

---

### 5. `src/4_parse_html.py`

功能：解析本地 HTML 提取补充字段，并与基础数据合并。

主要逻辑：

- 遍历 `html_cache/` 下全部类型目录
- 使用 BeautifulSoup 解析 HTML
- 提取以下字段：
  - `subject_id`
  - 电影名
  - 导演
  - 片长
  - 剧情简介
  - IMDb编号
- 读取第2步筛选后的中国地区电影数据
- 优先直接使用基础表中已有的 `subject_id`
- 只有在 `subject_id` 缺失时，才从 `详情链接` 兜底提取
- 优先按 `subject_id + 影片类型` 做左连接合并
- 如果个别记录没有 `subject_id`，再退回按 `电影名 + 影片类型` 合并
- 输出最终文件 `data/filtered/china_all.xlsx`

相比旧版本，现在第4步已经与第3步新的 HTML 命名方式同步：

- 第3步用 `subject_id_电影名.html` 保存
- 第4步会从文件名中提取 `subject_id`
- 合并时优先使用 `subject_id`，比单纯按电影名更稳定

最终数据会在基础信息上补充导演、片长、剧情简介和 IMDb 编号字段。

---

## 四、完整执行流程

请按如下顺序执行：

```bash
python src/1_get_raw_data.py
python src/2_filter_china.py
python src/3_save_html.py
python src/4_parse_html.py
```

流程说明：

1. 第1步抓取三类电影在各个评分区间下的原始榜单数据，并统一去重
2. 第2步筛选其中属于中国地区（大陆 / 香港 / 台湾）的电影
3. 第3步下载这些电影的详情页 HTML，并按 `subject_id` 稳定命名
4. 第4步解析 HTML 并按 `subject_id` 优先合并生成最终总表

## 七、配置项说明

你可以在 `config.py` 中调整以下内容：

### 1. 电影类型配置

```python
MOVIE_TYPES = {
    "comedy": {"type_id": 24, "name": "喜剧"},
    "action": {"type_id": 5, "name": "动作"},
    "romance": {"type_id": 13, "name": "爱情"}
}
```

如果后续要增加类型，只需要追加新的映射。

### 2. 中国地区关键词

```python
CHINA_KEYWORDS = ["中国大陆", "大陆", "中国香港", "香港", "中国台湾", "台湾"]
```

用于筛选中国地区电影（大陆 / 香港 / 台湾）。

### 3. 请求延时

```python
DELAY_MIN = 3
DELAY_MAX = 5
```

用于控制下载 HTML 时的随机等待时间，降低请求过快带来的风控风险。

### 4. Cookie 与请求头

Cookie 会定期失效，需要根据浏览器中的最新登录态进行更新。更新后只需要修改 `config.py` 中的 `COOKIES` 即可。

## 八、项目特点

本项目相比旧版脚本，具有以下改进：

1. **结构更清晰**：按配置、脚本、数据、缓存分目录组织
2. **支持多类型抓取**：不再局限于单一喜剧类型
3. **评分区间遍历更完整**：第1步会自动遍历 `100:90` 到 `10:0`
4. **支持断点续传**：HTML 已下载文件会自动跳过
5. **HTML 命名更稳定**：优先使用 `subject_id_电影名.html`
6. **合并更准确**：第4步优先按 `subject_id + 影片类型` 合并
7. **便于维护**：所有核心配置集中在 `config.py`
8. **流程标准化**：四步脚本职责明确，便于逐步执行和排错

## 九、注意事项

1. **Cookie 可能过期**  
   若抓取报错或返回异常，优先检查 `config.py` 中的 Cookie 是否失效。

2. **豆瓣榜单接口并非全站全量电影库**  
   当前第1步基于豆瓣分类榜单接口抓取。即使已经遍历了所有评分区间，得到的数据仍然是该接口可返回的范围，不等同于豆瓣全站所有该类型电影。

3. **Excel 文件不要在运行时打开**  
   如果 `data/raw/*.xlsx`、`data/filtered/*.xlsx` 或最终输出文件正在被 Excel 占用，脚本可能无法写入。

4. **HTML 依赖网络请求**  
   第3步包含随机延时、批次暂停和风控退避重试，耗时较长属于正常现象。建议运行时保持网络稳定。

5. **中国地区筛选已包含香港和台湾**  
   当前第2步不再只保留大陆电影，而是会保留 `中国大陆`、`中国香港`、`中国台湾` 相关影片。

6. **异常页面会被拦截**  
   第3步会校验页面是否为有效电影详情页。如果遇到验证码页、异常页、登录页、禁止访问页或结构不完整的页面，会判定为失败而不是直接保存。

7. **合并准确率已提升，但仍建议抽样检查**  
   当前第4步已优先按 `subject_id` 合并，准确率比单纯按电影名更高，但仍建议抽样检查最终结果中的 `导演`、`片长` 是否正常。

8. **依赖库**  
   项目运行依赖：
   - `requests`
   - `pandas`
   - `openpyxl`
   - `beautifulsoup4`

可使用如下命令安装：

```bash
pip install requests pandas openpyxl beautifulsoup4
```
