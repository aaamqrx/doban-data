# 豆瓣电影爬虫项目快速说明

## 一句话概括
6 步跑完整个流程：**抓原始数据 → 筛中国地区电影 → 下载 HTML → 缓存百度指数 → 解析百度指数 → 解析 HTML 并最终合并**。

---

## 运行顺序

```bash
python src/1_get_raw_data.py
python src/2_filter_china.py
python src/3_save_html.py
python src/4_save_baidu_index.py
python src/5_parse_baidu_index.py
python src/6_parse_html.py
```

如果是新环境，先执行：`pip install -r requirements.txt`。

---

## 每一步干什么

### 1. [src/1_get_raw_data.py](src/1_get_raw_data.py)
- 按类型抓豆瓣榜单
- 遍历评分区间 `100:90` 到 `10:0`
- 分页抓取后汇总
- 优先按 `subject_id` 去重
- 直接保存 `subject_id / 评分 / 评价人数`
- 输出：`data/raw/*.xlsx`

### 2. [src/2_filter_china.py](src/2_filter_china.py)
- 从原始数据里筛中国地区电影（大陆 / 香港 / 台湾）
- 保留 `subject_id / 评分 / 评价人数`
- 基于 `上映时间` 生成 `春节档 / 国庆档 / 五一档 / 暑期档 / 普通周末` 布尔特征
- 增加 `影片类型`
- 去掉原始 `类型` 列
- 输出：`data/filtered/china_*.xlsx`

### 3. [src/3_save_html.py](src/3_save_html.py)
- 读取 `china_*.xlsx`
- 下载详情页 HTML
- 文件名优先为：`subject_id_电影名.html`
- 自动跳过已下载文件
- 会拦截异常页、验证码页、登录页
- 命中“禁止访问”等风控页会自动退避重试，并在连续拦截时暂停/中止
- 输出：`html_cache/<类型>/`

### 4. [src/4_save_baidu_index.py](src/4_save_baidu_index.py)
- 读取 `china_*.xlsx`
- 批量前用 [config.py](config.py) 里配置的预检词与日期窗口测一次百度指数，失败则直接退出
- 上映前 7 天窗口早于 `BAIDU_INDEX_MIN_DATE` 时跳过请求，写占位缓存（该日期为经验阈值，可按实测改）
- 计算上映前 7 天日期窗口
- 清洗百度查询词，并按「清洗后 → 原始片名」尝试抓取
- 抓取并缓存百度指数日值 JSON（含多种诊断状态，详见 [项目说明.md](项目说明.md) 第 4 步）
- 输出：`baidu_cache/<类型>/`

### 5. [src/5_parse_baidu_index.py](src/5_parse_baidu_index.py)
- 读取 `data/filtered/china_*.xlsx`，逐行匹配 `baidu_cache/<类型>/` 下 JSON
- 生成 `上映前7天百度搜索指数均值`（终态失败类缓存不填均值，仅保留状态列）
- 保留 `百度指数状态 / 错误信息 / 查询词 / 样本天数 / 非空天数`
- 输出：`data/baidu/china_*.xlsx`

### 6. [src/6_parse_html.py](src/6_parse_html.py)
- 解析 HTML 中的 `电影名 / 导演 / 片长 / 剧情简介 / IMDb编号`
- 从 `data/baidu/china_*.xlsx` 读取基础表
- 优先按 `subject_id` 合并
- 输出：`data/final/china_all.xlsx`

## 常见注意事项

1. **Cookie 失效就会出问题**  
   豆瓣和百度指数都依赖 [config.py](config.py) 中的 Cookie。

2. **第2步档期特征依赖 `chinesecalendar`**  
   报导入错误时先执行：`pip install -r requirements.txt`。

3. **第4步依赖 `qdata`**  
   如果运行百度指数脚本报模块错误，也先执行：`pip install -r requirements.txt`。

4. **第4步预检失败**  
   脚本一启动会先拉一次预检窗口；若失败，先更新 [config.py](config.py) 中百度 Cookie，或调整预检关键词与日期，再检查网络。

5. **不要打开 Excel 再运行脚本**  
   否则 `data/raw/`、`data/filtered/`、`data/baidu/` 或 `data/final/` 下的写文件可能失败。

6. **第3步和第4步比较慢是正常的**  
   因为有随机延时、批次暂停和失败重试。

7. **百度指数全 0 不一定是失败**  
   成功抓到但 7 天都为 0 的记录会在第 5 步标记为 `zero_valid`。

8. **副标题会被清洗**  
   第 4 步会优先截断副标题，缓存里会保留原始查询词和清洗后查询词。

9. **最终建议抽样检查**  
   重点看 `上映前7天百度搜索指数均值`、`导演`、`片长` 是否正常。

## 当前这套流程的关键词

- 评分区间遍历
- 中国地区电影筛选
- HTML 断点续传
- 百度指数先缓存再解析
- `subject_id` 优先合并
- 第2步生成档期布尔特征
- 第5步生成百度热度特征
- 第6步补剧情简介与 IMDb 编号

这几个点就是当前项目最核心的逻辑。
