# 豆瓣电影爬虫项目快速说明

## 一句话概括
4 步跑完整个流程：**抓原始数据 → 筛中国地区电影 → 下载 HTML → 解析并合并**。

---

## 运行顺序

```bash
python src/1_get_raw_data.py
python src/2_filter_china.py
python src/3_save_html.py
python src/4_parse_html.py
```

---

## 每一步干什么

### 1. [src/1_get_raw_data.py](src/1_get_raw_data.py)
- 按类型抓豆瓣榜单
- 遍历评分区间 `100:90` 到 `10:0`
- 分页抓取后汇总
- 优先按 `subject_id` 去重
- 直接保存 `subject_id / 评分 / 评价人数`
- 输出：`data/raw/*.xlsx`

### 2. [src/2_filter_china.py](src/2_filter_china.py)
- 从原始数据里筛中国地区电影（大陆 / 香港 / 台湾）
- 保留 `subject_id / 评分 / 评价人数`
- 增加 `影片类型`
- 去掉原始 `类型` 列
- 输出：`data/filtered/china_*.xlsx`

### 3. [src/3_save_html.py](src/3_save_html.py)
- 读取 `china_*.xlsx`
- 下载详情页 HTML
- 文件名优先为：`subject_id_电影名.html`
- 自动跳过已下载文件
- 会拦截异常页、验证码页、登录页
- 命中“禁止访问”等风控页会自动退避重试，并在连续拦截时暂停/中止
- 输出：`html_cache/<类型>/`

### 4. [src/4_parse_html.py](src/4_parse_html.py)
- 解析 HTML 中的 `电影名 / 导演 / 片长 / 剧情简介 / IMDb编号`
- 从文件名提取 `subject_id`
- 优先直接使用基础表已有的 `subject_id`
- 仅在缺失时才从详情链接兜底提取
- 优先按 `subject_id` 合并
- 输出：`data/filtered/china_all.xlsx`

## 常见注意事项

1. **Cookie 失效就会出问题**  
   出现抓取异常先看 [config.py](config.py)。

2. **不要打开 Excel 再运行脚本**  
   否则写文件可能失败。

3. **第3步比较慢是正常的**  
   因为有随机延时、批次暂停和风控退避重试。

4. **榜单接口不是全站全量电影库**  
   即使遍历全部评分区间，数据量仍受豆瓣榜单接口限制。

5. **第2步当前包含大陆 / 香港 / 台湾**  
   现在筛选的是中国地区电影，不再只保留大陆电影。

6. **最终建议抽样检查**  
   重点看 `导演`、`片长` 是否正常。
